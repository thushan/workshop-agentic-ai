Foundations

Understanding the Building Blocks

AI has been on a journey

Life before Modern AI

  • Rule-based systems - if/else logic, brittle, hard to scale
  • Traditional NLP - tokenisation, regex, bag-of-words
  • Classic ML - models trained on features, narrow scope
  • Useful, but limited ‚Üí not conversational, not adaptive

Machine Learning Models

üìä Traditional ML

Decision Trees, Random Forests, Regression

Worked well on structured/tabular data like spreadsheets and metrics.

Narrow scope ¬∑ Feature engineering required

üß† Deep Learning

CNNs, RNNs, Neural Nets

Powerful for images, speech, sequences. Set the stage for transformers.

Enabled unstructured data ‚Üí paved way to LLMs

Objects To Vectors

How models convert objects into vectors

We can unify all modalities into one architecture...

Transformers + multimodal

  • One unified architecture - same model, many tasks
  • Handles text, images, audio and video
  • Normalises to Vectors (Embeddings)

Demo: Characters can speak

What are LLMs?

  • Large Language Models predict the next token for a given context.
  • Great for Q&A, summarising, transforming text & code.
  • But by default: stateless and one-shot.
Keep it tight-audience are devs. The key idea is ‚Äústateless by default.‚Äù

Limitations of LLMs

  • Stateless - no memory beyond the prompt
  • No planning - one-shot answers only
  • No actions - inability to use external tools or APIs
  • Hallucinations - confident but wrong outputs

‚Üí Sets the stage for Agentic AI

Agentic Concepts

How Agents Think and Act

What is an Agent?

An agent is an AI system that decides, acts and learns in a loop - not just one-shot.

Agents combine LLM reasoning with memory and tools.

Tooling

üìä Database

Query internal program data (bios, feedback, activity logs, skills, goals).

üìÖ Calendar / Email

Send invites, reminders, or schedule mentoring sessions.

üîç Search

Look up knowledge base articles or policies to support mentoring.

üåê APIs

Internal / External APIs, MCP Servers, Services - Slack

ü§ñ Tools let agents act, not just generate text.

üí° Remember Claude Code refactoring code?

Agent Memory

üìù Short-Term Memory

Keeps track of the current conversation or task

Similar to an LLM context window

 

Examples: remembering last mentor searched, current filters, conversation state

üìö Long-Term Memory

Persists knowledge across sessions

Can be stored in databases, embeddings or logs

 

Examples: past mentor/mentee interactions, program history, feedback logs

Short what a mentor remembers from this session

Long the mentee's entire journey/history across the program

Planning

Agents can break a big goal into smaller sub-tasks.

MentorLoop Example

'Boost engagement' ‚Üí identify inactive mentees ‚Üí draft nudge ‚Üí send message

ReAct Pattern

Reason + Act steps for lightweight planning

We're giving it space to think out-loud between actions.

üß† Reason

LLM "thinks aloud" step-by-step

e.g. I need mentee's goals first

‚öôÔ∏è Act

Call a tool or API, observe result

e.g. fetch mentee profile from MongoDB
Just in case

We're not talking React, the Javascript library!
